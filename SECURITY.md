# Security Policy

## Reporting Security Vulnerabilities

The ARES project takes security seriously. We appreciate your efforts to responsibly disclose your findings.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: security@[project-domain].org

You should receive a response within 48 hours. If for some reason you do not, please follow up via Twitter @ChooChoo69420.

Please include:
- Type of issue
- Full paths of source file(s) related to the issue
- Location of affected source code
- Step-by-step instructions to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact of the issue

## Preferred Languages

We prefer all communications to be in English.

## Policy

We follow the principle of Coordinated Vulnerability Disclosure.

## Safe Harbor

Any activities conducted in a manner consistent with this policy will be considered authorized conduct and we will not initiate legal action against you. If legal action is initiated against you in connection with activities conducted under this policy, we will take steps to make it known that your actions were conducted in compliance with this policy.
